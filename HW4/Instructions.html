American Association of Community Colleges Home Page Requirements (100 Points)

To start, look at the American Association of Community Colleges home page. Then, open up the source code and take a look at it. If you look carefully, you'll see that the layout of the page is done using tables. Different page components are placed into table cells, which dictate where on the screen they will appear. In Chapter 6 and 7, we discussed another mechanism for laying out pages using style sheets.

Recreate the American Association of Community Colleges home page (1 page only) with all the same content and layout such that it adheres to the following requirements:

  • It must use CSS positioning for all visible Web page components. That means you should break up the page into different entities, then each will be placed into their own <div> elements which can be positioned in the style sheet.
  • You may not use tables for any purpose in this assignment. The first thing the TA will do is make sure you didn't violate this rule.
  • Try to recreate an identical page layout as best you can using this new CSS positioning technique. This means you will want similarly sized, similarly placed page components. Make sure all components are properly aligned. Ragged alignment in Web pages makes for very poor style. You may take as much code directly from the original site as you like.
  • Try to simulate all the same page style as in the original page. This would include:
    1. Formatted Hyperlinks, which should also all work
    2. Colors
    3. Fonts

NOTE: Don't break your back trying to get an exact replica of the original site with all the same dimensions, colors, fonts, etc ..., but try to approximate the same look. Points will be deducted for sites with significant differences.

NOTE: Your site should not include any JavaScript taken from the original American Association of Community Colleges home page, and so it will not have to generate anything dynamically. Just choose a moment in time from the home page and use those images, links, etc ...

NOTE: it is important that you use your style sheet for all presentation specifications. I continue to stress this as this is the professional way of doing things.

Bonus Requirements (+10 Points)

If you look at the American Association of Community Colleges home page, you'll see there is a form at the bottom of the screen. It's there for searching the site. Instead of using that form, for bonus points, your Web page should allow a user to enter a message into a text field and an email address into another text field, and then send off the email by pressing a button. This would trigger a response page to the Web surfer. To do this, your page should do the following:

  • Use 2 Text Field Controls - one should be titled "Message", and one titled "Email". You should make these the same size, and make sure they fit well with your layout. Remember, surfers like components to be nice and neat and aligned in a page, so you should make sure your page meets this criteria.
  • Send Button - when someone enters a message and email address and presses this button, it should activate your script. This will be a different script than that used by the American Association of Community Colleges site's form, so make sure you specify the proper action for your form. This "Send" button will function as our submit button.
  • The CGI Script - I have provided email.cgi, an initial version of a script you will have to modify to get the correct behavior. Make sure you don't forget the .cgi file extension when saving this file. It is essentially the join.cgi script from the textbook, which itself sends an email (we'll discuss how in class). Once you change the script, it should do the following:
    1. Check to make sure a message and email address were entered. It doesn't have to check to make sure the email address is real, just make sure text was entered in the field. If either field is left blank, the script should return a Web page instructing the user to enter the required information.
    2. When text is entered in both fields, the program should do 2 things:
      1. It should send an email to the email address using the message provided by the user
      2. It should return a Web page to the user simply informing him/her of the email, including who it was sent to and what the message said.

In addition, as you have noticed, the American Association of Community Colleges home page has a slide show section at the top of the page. This is done by JavaScript inside the html code. Try to modify the JavaScript so that the page displays a picture slide show with the pictures chosen by you.

NOTE: just so you know, not many changes need to be made to the CGI file, but they will require that you understand some of the basics of what the program is doing. Don't be afraid, it's not terribly complex. Examining the file you could probably figure out most of what it's doing.

NOTE: after placing the .cgi program in your Sparky account's cgi-bin directory, don't forget to set the file permissions to 755.
